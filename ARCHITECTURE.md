# ReadYou Architecture

ReadYou is a single-module Android application written in Kotlin and Jetpack Compose.

## Build

- Root Gradle project: `settings.gradle.kts`, `build.gradle.kts`
- App module: `app`
- Main variant used locally: `githubDebug`
- Package name: `me.ash.reader`
- Main activity: `me.ash.reader.infrastructure.android.MainActivity`
- Dependency management: Gradle version catalog in `gradle/libs.versions.toml`

## Main Layers

### Domain

Path: `app/src/main/java/me/ash/reader/domain`

This layer contains business-facing data contracts and RSS logic:

- `model`: Room entities and domain models for accounts, groups, feeds, articles, filters, and thought data.
- `repository`: Room DAO interfaces for account, group, feed, and article persistence.
- `service`: RSS synchronization, OPML import/export, Gemini integration, workers, and account service abstractions.
- `data`: app use cases and flow composition, such as paging article lists, filter state, grouped feed lists, and sync logging.

### Infrastructure

Path: `app/src/main/java/me/ash/reader/infrastructure`

This layer hosts platform and integration code:

- `android`: `AndroidApp`, `MainActivity`, crash reporting, notifications, TTS, and Android entry behavior.
- `db`: Room database setup and migrations.
- `di`: Hilt modules for database, OkHttp, Retrofit, workers, coroutine scopes/dispatchers, image loading, settings, and use cases.
- `rss`: RSS parsing and provider implementations, including local RSS, Fever, FreshRSS, and Google Reader-compatible APIs.
- `net`, `html`, `tts`, `datastore`, `preference`: networking helpers, article rendering/parsing support, text-to-speech, DataStore, and typed preference models.

### UI

Path: `app/src/main/java/me/ash/reader/ui`

The UI is Compose-based:

- `page/nav3`: Navigation 3 route keys and app navigation entry.
- `page/home`: feed list, article flow, reading page, drawers, subscription flows, and related view models.
- `page/settings`: account, color/style, interaction, language, troubleshooting, support, Gemini, and TTS settings.
- `page/startup`: first-run welcome/onboarding.
- `component`: reusable Compose controls, menus, reader components, scrollbars, swipe handling, and webview wrappers.
- `theme`: app typography, theme, shapes, and dynamic color/palette implementation.
- `widget`: Glance app widgets and widget configuration activity.

## Runtime Flow

1. `AndroidApp` initializes Hilt and application-level setup.
2. `MainActivity` chooses the initial route from first-launch state and incoming intents.
3. `AppEntry` owns the Navigation 3 back stack and maps `Route` keys to Compose pages.
4. View models and use cases read/write through Room DAOs, DataStore preferences, RSS services, and workers.
5. Sync work is performed through WorkManager workers and RSS provider implementations, then persisted into Room.
6. Compose pages observe flows/paging data and render the feed, article list, reader, settings, and startup screens.

## External Dependencies

Key libraries include AndroidX Compose, Navigation 3, Material 3 adaptive components, Room, Paging, WorkManager, Hilt, Retrofit, OkHttp, Coil, Glance widgets, Kotlin serialization, ROME RSS parsing, readability4j, and Google Generative AI.
